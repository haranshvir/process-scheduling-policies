Process Scheduling Overview


Click on the play buttons within each of the three visuals (you can select non-stop play for a seamless effect) to see how Process Scheduling happens.
Key Features: Each process is executed in the order of its arrival. As the name suggests, each process is executed in a first come, first served manner. This is easy to understand and implement, but has no benefits. Results in poor average wait time and throughput.
Key Features: Here, the processor checks the process queue and executed the process having the shortest burst time first, followed by the second shortest burst time, and so on. Hence the name shortest job first. Larger processes are executed towards the end, which results in optimal average waiting time for all processes.
Key Features: Each process runs for a fixed amount of time called Time-quantum. Once a process is executed for one quantum, it is preempted and other process executes. This process continues till all the processes are completed. Lower time-quantum leads to more context switches, higher time-quantum reduces it to FCFS.
